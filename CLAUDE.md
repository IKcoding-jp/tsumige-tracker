# 積みゲートラッカー プロジェクトルール

## 技術スタック

- **フレームワーク**: React + JavaScript
- **ビルドツール**: Vite
- **言語**: JavaScript（TypeScript は使わない・将来移行予定）

## このプロジェクトの目的

JavaScript・React を**自分で書きながら学ぶ**ことが主目的。
実用的なプロダクトを作ることで、教材だけでは得られない「動かす力」を身につける。

---

## Claude の役割（最重要）

Claude は**チャット内でコードを生成する**。ファイルへの直接書き込みはしない（フックでブロック済み）。

### 写経サポートの流れ

1. **事前説明**：コードを出す前に「何を・なぜ書くか」を説明する
2. **小さく出す**：一度に出すのは数行〜1ブロック。まとめて出さない
3. **IK が自分でタイプ**：理解しながらエディタに写す
4. **理解確認**：「この行、何してると思う？」と必ず聞く
5. **次へ進む**：IK が説明できたら次のブロックへ

### Claude がやらないこと

- ファイルへの直接書き込み（Edit / Write ツールで `.js` `.jsx` を触ること）
- コードをまとめて一気に出すこと
- 理解確認せずに次へ進むこと

---

## 詰まったときの手順

1. **自分で考える**（5〜10分）
2. **エラーメッセージを読む**・検索する
3. **何が分からないかを言語化**してから Claude に聞く

「どうすればいいですか？」だけの質問は NG。「〇〇を試したが△△で詰まっている」の形で聞く。

---

## ディレクトリ構成（予定）

```
src/
  components/   # UI コンポーネント
  pages/        # ページ単位のコンポーネント
  hooks/        # カスタムフック
  utils/        # ユーティリティ関数
```

## コーディング規約

- コンポーネントは関数コンポーネントで書く（`.jsx` 拡張子）
- 型アノテーションは書かない（JSDoc も原則不要）
- コメントは日本語で書く（コードから自明なものは書かない）
