# 積みゲートラッカー 設計仕様書

作成日: 2026-06-28

## 概要

積んだゲームの一覧を管理するWebアプリ。
JavaScript・React の学習を兼ねた実用ポートフォリオとして開発する。

---

## 技術スタック

| 項目 | 内容 |
|---|---|
| フロントエンド | React + JavaScript |
| ビルドツール | Vite |
| バックエンド | Supabase（PostgreSQL + Auth）|

---

## データ構造

### games テーブル

| カラム | 型 | 内容 |
|---|---|---|
| id | uuid | 主キー・自動生成 |
| user_id | uuid | 認証ユーザーのID（外部キー）|
| title | text | ゲームタイトル |
| platform | text | Switch / PS5 / PC など |
| status | text | backlog / playing / completed |
| rating | integer | 1〜5・未評価は null |
| priority | integer | 1（高）/ 2（中）/ 3（低）|
| memo | text | 自由テキスト |
| created_at | timestamp | 登録日・自動生成 |

RLS（Row Level Security）により、ログインユーザーは自分のレコードのみ参照・編集可能。

---

## 画面構成

```
ログイン画面
  ↓ ログイン成功
一覧画面（メイン）─── 追加ボタン → 追加モーダル
  │
  └─ ゲームをタップ → 編集モーダル
```

### ログイン画面
- メールアドレス＋パスワードで認証
- 新規登録も同じページで行う

### 一覧画面（メイン）
- ゲームがカード形式で一覧表示
- ステータス（すべて・積み中・プレイ中・クリア済み）でフィルタリング可能

### 追加・編集モーダル
- 別ページではなくモーダル（画面上に重ねて表示）
- 全フィールド（タイトル・プラットフォーム・ステータス・評価・優先度・メモ）を入力

---

## コンポーネント構成

```
App
├── LoginPage（ログイン画面）
└── GameListPage（一覧画面）
    ├── Header（タイトル＋ログアウトボタン）
    ├── FilterBar（ステータスで絞り込む）
    ├── GameCard（ゲーム1枚のカード表示）
    └── GameModal（追加・編集フォーム）
```

各コンポーネントは1つの責任だけを持つ。
`GameCard` はカードを表示するだけ、`FilterBar` は絞り込みだけ、という設計。
